My first blog

这是我的第一篇在github上的博客文章 😂

在这片文章中我将要记录一下如何使用hexo配合github在公网中建立自己的博客网站。

1.拥有一个github的账户

github是世界上最大的同性交友网站，它的注册是免费的。对于国内大多数人可能需要科学上网才能畅通注册，我在这里不过多交流。八仙过海各凭本事。

创立账户之后还要配置本地git与github之间的密钥来完成ssh通信，因为github貌似不支持https进行上传了（不清楚），总之使用ssh非常方便。配置密钥网上有很多教程，这里不过多赘述。

2.在github上建立一个新仓库

在主页点击左上角的New按钮

在Repository name下输入名字：你的用户名.github.io。
假设你的名字是123，那么这个仓库的名字就应该是123.github.io。

Description用来描述仓库，是可选选项。

不要选择private如果你想让别人看到你的博客的话，只能选择public

选上Add a README file。

最后create就创建完成了

3.下载hexo工具

hexo是一个基于nodejs的工具，所以需要先安装nodejs的环境。

运行npm --version正常输出版本表示安装成功。

执行npm install hexo-cli -g全局安装hexo。

在本地电脑上选择或者新建一个用于写博客的目录，在目录中执行hexo init初始化项目，再执行npm install安装依赖。这时在目录下就会出现很多文件和文件夹。

我们用于写博客文章的地方就在source/_posts目录下，source目录同时可以放入一些图片等需要使用的资源。

在public目录下放入的是hexo工具为我们写出的md生成的前端文件（html、css、js）。

通过hexo help我们能看到hexo的常用操作。

我们使用hexo generate生成前端代码，使用hexo server启动服务，点击链接或者打开浏览器访问指定的端口。网页上有一个Hello World的博客，对应source/_posts目录下的一篇md文件。

使用hexo new 文件名新建博客文章

更多配置文件和使用参考官网

4.修改配置文件

找到目录下的_config.yml文件，里面最后一行有deploy，将其改为

branch应该看github创建的仓库的default分支名称，2024/10/5年默认为main。

最终我们使用hexo deploypush到指定仓库中可以使用 https://用户名.github.io 访问了。😁
